# Go HAL Component Template

This is a template for building Go-based HAL userspace components for LinuxCNC.

## Prerequisites

- Go 1.21 or later
- LinuxCNC installed, or RIP (Run-In-Place) build with environment sourced

## Quick Start

### 1. Copy this template

```bash
cp -r /usr/share/linuxcnc/hal-go-template ~/my-hal-component
cd ~/my-hal-component
```

### 2. Build

**With installed LinuxCNC:**
```bash
make
```

**With RIP build:**
```bash
source /path/to/linuxcnc/scripts/rip-environment
make
```

### 3. Install (Optional)

To install the component to the LinuxCNC bin directory:

```bash
make install
```

This will:
- For RIP builds: Install to `$EMC2_HOME/bin`
- For installed LinuxCNC: Install to the same directory as `halcompile`

To uninstall:

```bash
make uninstall
```

### 4. Test

```bash
halrun
halcmd: loadusr -W ./mycomponent
halcmd: show pin mycomponent.*
halcmd: exit
```

## Customizing

1. Edit `main.go` to implement your component logic
2. Change the component name in `main.go`: `hal.NewComponent("mycomponent")`
3. Update `COMPONENT` in `Makefile` to match your component name
4. Update `module` line in `go.mod` if desired

## Project Structure

```
my-hal-component/
├── Makefile    # Build system
├── go.mod      # Go module definition  
├── go.work     # Go workspace (auto-generated by make)
└── main.go     # Your component code
```

## How It Works

The Makefile:
1. Detects LinuxCNC installation (RIP or system-installed)
2. Locates the hal-go package
3. Creates a Go workspace to reference hal-go
4. Builds with proper CGO flags for HAL library

## Troubleshooting

**"hal-go package not found"**
- Ensure LinuxCNC is installed, or source `scripts/rip-environment` for RIP builds

**"cannot find -llinuxcnchal"**
- LinuxCNC libraries not in path. For RIP: source `scripts/rip-environment`
- For installed: check that `liblinuxcnchal.so` exists in `/usr/lib/linuxcnc/`

**Component hangs on load**
- Ensure component name matches binary name
- Check that `comp.Ready()` is called after pin creation
